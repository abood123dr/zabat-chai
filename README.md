# ☕ ظبط شاي - Zabat Chai

نظام إدارة مقاهي ومطاعم متقدم مع دعم **SaaS متعدد المستأجرين** وتطبيق موبايل محسّن.

## ✨ الميزات

### 🏢 نظام SaaS متعدد المستأجرين
- **Super Admin**: إنشاء وإدارة عدة مقاهي/مطاعم
- **عزل البيانات**: كل عمل له بيانات منفصلة تماماً
- **إدارة المستخدمين**: تعيين موظفين لكل عمل بأدوار مختلفة
- **نموذج الاشتراك**: جاهز للتوسع والترخيص المستقبلي

### 📱 واجهات ذكية
- **لوحة تحكم**: إحصائيات المبيعات والطلبات
- **نظام الكاشير**: إدارة الطلبات والدفع بسهولة
- **قائمة العملاء**: استعراض المنتجات وتقديم الطلبات
- **متابعة الطلبات**: عميل يتابع طلبه في الوقت الفعلي
- **إدارة المنتجات**: إضافة/تعديل/حذف المنتجات والفئات
- **إدارة الطاولات والغرف**: نموذج غرف الألعاب (gaming rooms)

### 🔐 نظام الأدوار والصلاحيات
- **Super Admin**: إدارة كل شيء (أعمال، مستخدمين، إعدادات)
- **Admin**: إدارة عمل واحد وموظفيه
- **Cashier**: معالجة الطلبات والدفع
- **Staff**: موظف عام

### 📊 المميزات الإضافية
- **إشعارات فورية**: تنبيهات صوتية للطلبات الجديدة
- **إحصائيات مفصلة**: مبيعات يومية، أفضل منتجات، توزيع الحالات
- **ملاحظات العملاء**: طلبات خاصة ورسائل الموظفين
- **دعم اللغة العربية**: واجهة RTL كاملة

## 🛠️ المتطلبات

- Node.js 16+
- npm أو yarn
- Supabase (للقاعدة البيانات والمصادقة)

## 🚀 البدء السريع

### 1. استنساخ المستودع
```bash
git clone <repository-url>
cd zabat-chai-main
```

### 2. تثبيت المكتبات
```bash
npm install
```

### 3. إعداد متغيرات البيئة
أنشئ ملف `.env.local`:
```env
VITE_SUPABASE_URL=your_supabase_url
VITE_SUPABASE_ANON_KEY=your_supabase_anon_key
```

### 4. تشغيل التطبيق
```bash
npm run dev
```

التطبيق سيعمل على: `http://localhost:5173`

## 📦 المكتبات المستخدمة

**Frontend:**
- **React 18**: واجهة المستخدم
- **React Router v6**: التوجيه والملاحة
- **Vite**: بيئة التطوير السريعة
- **TailwindCSS**: تصميم الواجهات
- **Shadcn UI**: مكونات واجهة محسّنة
- **Lucide React**: أيقونات عالية الجودة
- **Recharts**: رسوم بيانية وإحصائيات

**Backend & Auth:**
- **Supabase**: قاعدة بيانات وMصادقة
- **React Query**: إدارة البيانات والـ API

## 🔄 دورة العمل الرئيسية

### Super Admin
1. إنشاء عمل جديد (مقهى/مطعم)
2. إضافة مستخدمين وتعيينهم لأدوار
3. عرض إحصائيات جميع الأعمال

### Admin (صاحب العمل)
1. تسجيل الدخول
2. إدارة المنتجات والفئات
3. إدارة الطاولات والغرف
4. عرض إحصائيات العمل

### Cashier (الكاشير)
1. عرض الطلبات القادمة
2. تحديث حالة الطلب (جاري التحضير → جاهز → تسليم)
3. إدارة غرف الألعاب والمدة الزمنية

### Customer (العميل)
1. عرض قائمة المنتجات
2. إضافة للسلة وتقديم طلب
3. متابعة حالة الطلب
4. طلب موظف أو طلب الحساب

## 🗄️ قاعدة البيانات

جداول Supabase:
- `users` (من Supabase Auth)
- `businesses` - المقاهي والمطاعم
- `business_users` - ربط المستخدمين بالأعمال
- `products` - المنتجات والمشروبات
- `categories` - فئات المنتجات
- `dining_tables` - الطاولات والغرف
- `orders` - الطلبات
- `room_sessions` - جلسات غرف الألعاب

## 🐛 استكشاف الأخطاء

### الاتصال برفضة Supabase؟
- تحقق من `VITE_SUPABASE_URL` و `VITE_SUPABASE_ANON_KEY` في `.env.local`
- تأكد من تفعيل RLS policies في Supabase

### الصفحات لا تحمل؟
- افتح console (F12) وابحث عن الأخطاء
- تأكد من تثبيت جميع المكتبات: `npm install`

### مشاكل الملاحة؟
- امسح cache: `npm run build` ثم جرب مجدداً
